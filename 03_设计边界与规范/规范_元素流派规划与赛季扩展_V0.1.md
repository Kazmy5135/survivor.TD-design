## 1. 规范目的

把元素 Build 从孤立技能配方提升为可长期扩展的流派体系。首发版本每个属性提供 3 个流派；后续赛季通过新神器、新技能表达和保留进化继续扩展，而不频繁修改属性公共语义。

## 2. 核心设计单位

### ARCHETYPE-001

- 等级：MUST
- 规则正文：元素流派规划的起点是“公共输入如何被转化为不同价值出口”，不是现有武器名称。

```text
元素公共输入
→ 流派价值出口
→ 神器拓扑
→ 技能职责与攻击形态
→ 具体武器表达
→ 武器进化
```

### ARCHETYPE-002

- 等级：MUST
- 规则正文：同属性首发 3 个流派应共享公共输入和部分输入型神器，并在价值出口层形成明确差异。
- 目的：让同属性神器具有交叉价值，同时避免三套流派只是同一套路的数值版本。

### ARCHETYPE-003

- 等级：MUST
- 规则正文：初始版本火、电、冰各规划 3 个正式流派。

## 3. 目标先行与三条控制轴

本项目采用“总目标 → 中间模型 → 模块落地 → 反向校验”的顺序。

### 3.1 单局时间轴

| 阶段 | 首关时间 | 设计目标 |
|---|---:|---|
| 启动期 | 0～3 分钟 | 至少一个技能或单件神器能建立可理解方向，不依赖唯一组件 |
| 成型期 | 3～6 分钟 | 两个技能或第一次进化形成基础分工，玩家能说出本局流派 |
| 收口期 | 6～8 分钟 | 第二件神器、第二次进化或关键被动形成胡牌/大胡候选 |
| 长局收口 | 8～12 分钟 | 为六技能阶段和更完整组合提供体验时间 |

以上阶段来自首关 Boss 节点和单局目标时长的设计推导，具体成长落点仍需概率模型验证。

### 3.2 Build 完成度轴

| 完成度 | 目标 | 相对无流派基线的有效效率候选 |
|---|---|---:|
| 基础成长 | 默认进化可正常战斗 | 1.00 |
| 乞丐版 | 单技能/单神器方向成立 | 1.10～1.25 |
| 基础成型 | 两技能形成分工 | 1.25～1.45 |
| 胡牌 | 两件神器与进化吻合 | 1.45～1.70 |
| 大胡 | 2～3 技能、被动和 2～3 神器完整吻合 | 1.70～2.00 |

这些倍率是建立数值模型的候选目标，不是最终战斗倍率。有效效率必须包含目标数、覆盖率、触发率和能耗。

### 3.3 赛季轴

- 首发：建立属性公共输入和 3 个基础价值出口。
- 后续赛季：深化现有出口、桥接两个出口、开启新出口或激活保留进化。
- 新神器可以形成新的最优组合，但旧 Build 的乞丐版和正常通关能力不得因此失效。

## 4. 中间模型

下钻技能和神器前必须先建立以下中间表：

1. **属性公共输入表**：状态、叠层、命中、能量等公共语言。
2. **价值出口表**：持续、消耗、利用、转化等不同兑现方式。
3. **神器拓扑表**：输入共享神器、双流派桥接神器、流派收口神器。
4. **技能职责表**：挂层、覆盖、传播、消耗、利用、资源桥接。
5. **攻击形态表**：单体/范围、线/扇形/圆形/弹道、瞬时/持续、频率与能耗。
6. **输出预算表**：直接伤害、持续伤害、触发伤害、资源收益和生存收益。
7. **进化覆盖表**：当前纳入、可兼容但未纳入、明确留给未来。

## 5. 神器拓扑规则

### ARTIFACT-TOPOLOGY-001

- 等级：MUST
- 规则正文：神器共享优先发生在“建立公共输入”阶段；流派差异优先发生在“如何兑现输入”阶段。

### ARTIFACT-TOPOLOGY-002

- 等级：SHOULD
- 规则正文：一个属性首发神器池应同时包含：
  - 支持 3 个流派的公共输入神器；
  - 支持其中 2 个流派的桥接神器；
  - 明确强化单一价值出口的收口神器。

### ARTIFACT-TOPOLOGY-003

- 等级：FORBIDDEN
- 规则正文：不得因为追求神器重叠，把流派收口神器也做成对所有路线等强，导致所有 Build 使用同一组最优神器。

## 6. 技能职责先于技能表现

### SKILL-ROLE-001

- 等级：MUST
- 规则正文：先确定技能在 Build 中承担的职责、攻击方式、目标规模、频率和能耗，再选择火焰喷射器、力场、导弹等表现。
- 边界：这里的技能名称只用于举例，不表示已经进入 8 个首发技能或 12 个长期技能名单。

### SKILL-ROLE-002

- 等级：MUST
- 规则正文：完全体 Build 的 2～3 个技能必须有分工，不能只是三个都会挂同一种状态的同质 AOE。

可用职责包括：

- 高频挂层器；
- 广域播种器；
- 持续覆盖器；
- 状态传播器；
- 全量消耗器；
- 状态利用器；
- 单体/Boss 收口器；
- 能量或跨属性桥接器。

## 7. 武器进化覆盖与留白

### EVOLUTION-001

- 等级：MUST
- 规则正文：首发三流派应覆盖大部分相关武器进化，但不能覆盖全部进化。

### EVOLUTION-002

- 等级：SHOULD
- 规则正文：首轮规划可将相关进化池的约 60%～75% 纳入完整 Build，保留约 25%～40% 暂不归属；该比例为设计候选。

### EVOLUTION-003

- 等级：MUST
- 规则正文：保留进化必须自身可玩并有基础数值价值，只是不进入当前三套理论最优组合；不得把留白做成明显废选项。

### EVOLUTION-004

- 等级：SHOULD
- 规则正文：后续赛季优先用新神器激活保留进化，而不是持续给已完整覆盖的路线叠加同类数值。

## 8. 数值预算规则

### VALUE-001

- 等级：MUST
- 规则正文：每个 Build 先确定输出桶占比，再给具体技能和神器分配倍率。

输出桶至少包括：

- 技能直接伤害；
- Debuff 持续伤害；
- 引爆或规则触发伤害；
- 能量收益；
- 控制与生存收益。

### VALUE-002

- 等级：FORBIDDEN
- 规则正文：同一份状态价值不能同时按完整持续伤害、完整引爆伤害和完整直接增伤重复计入。

### VALUE-003

- 等级：MUST
- 规则正文：AOE 数值不能用场上 400～500 个敌人直接相乘，必须使用期望有效目标数、攻击范围、目标密度、命中上限和有效覆盖时间。

```text
有效输出效率 =
（单目标伤害 × 期望有效目标数 × 有效覆盖率 × 触发成功率）÷ 实际能量成本
```

### VALUE-004

- 等级：MUST
- 规则正文：采用双向规划。
  - 自上而下：从单局阶段和 Build 完成度目标拆到技能、被动和神器。
  - 自下而上：从技能命中、目标数、持续时间、状态层数和能耗反推整局结果。
  - 比较误差：先排查重复估值，再调整局部数值。

## 9. 赛季上位构筑规则

### SEASON-001

- 等级：MUST
- 规则正文：新赛季神器必须说明它服务的长期目标缺口：深化、桥接、新出口或激活留白。

### SEASON-002

- 等级：SHOULD
- 规则正文：上位构筑优先通过更完整的机制闭环或更匹配的新神器形成，不使用“相同条件、相同效果、倍率更高”的纯替换作为主要内容。

### SEASON-003

- 等级：MUST
- 规则正文：新上位构筑可以提高理论上限，但不得让旧构筑低于正常通关下限。

### SEASON-004

- 等级：MUST
- 规则正文：公共元素状态语义是长期中台。新增神器原则上扩展状态的价值出口，不随赛季反复修改燃烧、感电或寒冷的基础定义。

## 10. 正式输出要求

- 元素合集先输出流派地图、神器拓扑、技能职责、进化覆盖和赛季接口。
- 每个具体 Build 再使用 [正式 Roguelike Build 模板](../00_Agent输出规范/模板_正式RoguelikeBuild文档.md) 单独成文。
- 具体 Build 必须列出事实、设计推导、候选和待验证数值。

## 11. 验收清单

- [ ] 先确定价值出口，再确定技能和神器。
- [ ] 首发每个属性恰有 3 个流派。
- [ ] 三个流派共享部分输入神器，但收口不同。
- [ ] 每个完全体有 2～3 个不同职责的主动技能。
- [ ] 没有覆盖全部相关武器进化。
- [ ] 保留进化本身可玩，不是废选项。
- [ ] 已建立乞丐到大胡的候选强度阶梯。
- [ ] 已拆分输出桶，未重复估值。
- [ ] 已说明新赛季神器如何形成上位构筑或新出口。
- [ ] 已进行自上而下和自下而上双向校验。
