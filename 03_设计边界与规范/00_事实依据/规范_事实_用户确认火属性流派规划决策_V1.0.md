# 用户确认的火属性流派规划决策 V1.0

> 文档类型：设计边界与规范  
> 事实等级：F0  
> 状态：Review  
> 确认者：设计负责人  
> 记录者：Codex  
> 创建日期：2026-07-21  
> 更新时间：2026-07-21  
> 适用范围：正式 Build 模板、首发属性流派数量、火属性流派总纲、技能进化覆盖与赛季扩展

---

## 1. 正式 Build 输出形式

- `Build_火_灼烧爆破_V0.2` 作为后续所有具体 Build 的基础模板。
- 后续正式 Build 必须采用同等级的信息完整度，至少包含事实依据、信息边界、状态与结算、主动技能进化、被动、神器、核心循环、乞丐/胡牌/大胡、非唯一解、数值接口和验收标准。

## 2. 首发流派数量

- 初始唯一版本中，火、电、冰各有 3 个首发 Build 流派。
- 当前阶段先从宏观层面完成火属性 3 个流派的规划，再下钻具体技能与神器。

## 3. 火属性三流派关系

- 三个火属性流派应共享部分神器，但不能只是同一套路的数值高低版本。
- 三个流派合计应覆盖大部分火属性相关武器进化，但不要求也不允许把全部进化都强行纳入现有 Build。
- 必须保留部分暂不属于任何完整 Build 的武器进化，为后续内容、玩家期待和赛季扩展留出空间。

## 4. 长线与赛季扩展

- 项目按商业化和赛季制长期更新考虑。
- 后续赛季会加入新神器。
- 新神器可以改变现有最优组合，并形成现有 Build 的上位构筑或新的价值出口。
- 设计首发 Build 合集时必须先建立长期指导纲领，避免每赛季只做孤立神器或纯数值叠加。

## 5. 火属性候选价值出口

设计负责人提出以下方向作为火属性宏观规划的重要候选：

- 叠加燃烧，主要依靠燃烧持续伤害；
- 叠加燃烧，但燃烧自身伤害较低，主要通过引爆燃烧造成瞬间伤害；
- 叠加燃烧，但燃烧自身伤害较低，主要提高对燃烧敌人的直接伤害；
- 叠加燃烧并把燃烧转化为能量，为其他技能或其他属性提供资源。

前三项可用于首发三流派候选；能量和跨属性方向可以作为后续扩展候选。具体流派定案属于派生设计，不属于本事实文档。

## 6. 设计顺序

火属性 Build 合集必须按以下顺序设计：

1. 确定流派方向和长期价值出口；
2. 确定不同流派之间共享与专属的神器类型；
3. 确定每个流派需要的技能职责、攻击方式、范围和单体定位；
4. 再寻找具体技能表现，例如火焰喷射器、火焰力场或导弹；
5. 最后分解每个武器的进化路线，并保留未参与当前 Build 的路线。

不得从已有武器名称出发，反向硬凑三套流派。

## 7. 数值设计方法

- 火属性 Build 合集必须遵循《数值设计思路总结》的方法：先定义阶段体验和总盘目标，再建立中间模型，之后拆到技能、被动和神器。
- 采用自上而下规划与自下而上反算相结合的双向校验。
- 不允许单个神器或技能局部看似合理，却在完整 Build、单局节奏或赛季长线上造成失衡。
