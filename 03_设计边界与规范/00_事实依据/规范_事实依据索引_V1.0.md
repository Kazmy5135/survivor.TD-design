## 1. 使用方式

本索引是 Agent 检查文档状态和定位候选事实的第一读取入口。

1. 先根据任务从下表定位事实来源。
2. 先检查状态；只有 Accepted 文档可以作为确定性前置。
3. 阅读对应本地原文或用户确认决策。
4. 再阅读派生规范和正式 Build。
5. 不从对话记忆补齐项目事实；对话中的新确认必须回写 Draft/Review，并经审核后成为 Accepted。

## 5. 冲突处理

当前已识别并完成收口的冲突：

| 主题 | 旧表述 | 当前口径 | 依据 |
|---|---|---|---|
| V1.2 地位 | 可作为当前方案继续细化 | 仅为参考候选 | F0 用户确认决策 |
| 元素 Debuff | 不可叠层 | 火、电、冰基础 Debuff 均可叠层 | F0 用户确认决策 |
| 持续伤害暴击 | V1.2 存在燃烧暴击入口 | 持续伤害不可暴击 | F0 用户确认决策 |
| 属性内部资源 | 使用热量、电荷、寒能 | 默认废弃，不创造只用于搬运数值的中间资源 | F0 用户确认决策 |
| 常驻技能 | 可能被理解为另一种技能形态 | 只是费用足够时自动重复下一次释放的便捷操作 | F0 用户确认决策 |
| 神器结构 | 两件指定神器形成基础循环 | 神器单件完整有效，多件联动属于胡牌上限 | F0 用户确认决策 |
| Debuff 来源 | 不同技能可能分别强化自己施加的状态层 | 同一种 Debuff 的所有层共享统一效果，不记录玩家不可识别的来源差异 | F0 用户确认决策 V1.1 |
| 中间层数条件 | 使用达到 5 层、消耗 5 层等条件 | 阈值只使用满层；消耗时清除全部当前层数 | F0 用户确认决策 V1.1 |
| 消耗后重挂 | 导弹消耗后立即补回基础燃烧 | 同一消耗事件结束时保持状态清除，不立即重挂 | F0 用户确认决策 V1.1 |
| 火焰力场定位 | 作为火属性首发候选技能参与 6/9 进化覆盖 | 暂不占用技能名额；仅作为持续覆盖职责的表现示例，待三元素与首发 8 技能定位完成后再决定 | F0 元素流派与技能容量决策 V1.0 |
| 跨属性规划时机 | 跨属性神器全部延后 | 当前先规划主辅属性和桥接接口；具体属性反应、神器内容与数值仍延后 | F0 元素流派与技能容量决策 V1.0 |
| 元素身份 | 火只负责伤害、冰只负责减速/控制、电只负责响应 | 每个元素都必须包含主输出与自己的副方向；冰不能被降格为纯辅助属性 | F0 元素流派与技能容量决策 V1.1 |
| 最小伤害 | 计算后伤害可以为 0 | 合法伤害事件的最终有效伤害最小为 1；无效目标或明确免疫不适用 | Accepted D1；F0 D1 审核记录 |
| 感电触发后状态 | 电击触发后感电继续保留 | 直接伤害触发电击，按触发前层数计算；电击结算后清除全部感电 | Accepted D1；F0 D1 审核记录 |
| 感电保留例外 | 把感电保留误写成基础规则 | 仅神器可以显式把电击后清除改为保留；主动、进化和被动不得默认保留 | Accepted D1 V0.3；F0 D1 审核记录 V1.1 |
| 寒冷减速 | 每层寒冷继续提高减速 | 1 层至满层前 1 层均提供同一个固定减速；满层全部转化为冰冻 | Accepted D1；F0 D1 审核记录 |
| 寒冷减速扩展 | 神器或成长可能把减速改成随层数增长 | 明确禁止任何组件让寒冷减速随层数提高；只允许统一修改固定减速值 | Accepted D1 V0.3；F0 D1 审核记录 V1.1 |
| 神器改写寒冷 | 把寒冷满层冰冻视为永远不可改变 | 神器可显式取消冰冻转化、修改施加层数或层数上限，但不得覆盖寒冷固定减速禁区 | Accepted D1 V0.3；F0 D2.1 流派生态决策 V1.0 |
