# 游戏愿景

## 文档目标

定义本产品必须持续兑现的玩家承诺和体验判断标准。产品范围与版本边界以 [ProductScope.md](../../00_Project/ProductScope.md) 为权威来源。

## 产品承诺

在 8～12 分钟的手机碎片时间内，玩家守住屏幕下方防线，通过即时释放技能消灭自上而下推进的敌人，并在持续的 Perk 三选一中构筑一条有明确目标、达成后产生强烈爽感的局内 Build。

## 核心体验

1. **割草结果清晰**：技能释放后，玩家能立即从敌人死亡和战场变化中确认技能效果。
2. **技能使用有决策**：玩家需要围绕技能费用判断释放时机，而不是只观看自动战斗。
3. **Build 目标可追求**：三选一过程持续向玩家展示可形成的强化方向。
4. **Build 达成有跃迁**：关键目标完成后，战斗效率、清场能力或战斗方式出现明确变化。
5. **Build 路线有差异**：不同组合形成机制和体验上的区别，而不是同一路线的数值高低差。
6. **跨局成长可感知**：局外天赋提高后续局内 Build 上限，但不取代局内选择的重要性。

## 体验节奏

- 单局目标时长为 8～12 分钟，服务碎片时间。
- 战斗压力、技能释放、升级选择和 Build 达成必须在单局内形成多次反馈循环。
- 首个关卡时长为 8 分钟，并在第 3、6、8 分钟设置 Boss 遭遇。

## 设计约束

- 玩家始终拥有主动释放技能的参与感。
- 技能释放后的主要反馈来自割草结果。
- Perk 三选一必须支持目标感和 Build 多样性。
- 首版本不以关卡通过率作为产品成功标准。
- 商业化功能和天赋之外的局外系统不属于首版本范围。

## 验收标准

- [ ] 玩家能够用一句话说明自己本局正在追求的 Build 方向。
- [ ] 技能释放后，玩家能够立即识别其对敌群或战场的作用。
- [ ] 至少存在多条机制或战斗体验明显不同的 Build 路线。
- [ ] 达成关键 Build 目标时出现明确的强度跃迁和爽感反馈。
- [ ] 一局能够在目标时长内完成完整的战斗、构筑与结算体验。
