# 核心战斗循环

## 文档目标

定义单局战斗中的防守目标、敌人推进、技能费用与释放、击杀成长、波次推进、Boss 遭遇、胜利和失败规则。

上游体验目标见 [GameVision.md](../00_Vision/GameVision.md)，产品范围见 [ProductScope.md](../../00_Project/ProductScope.md)。

## 功能范围

### 包含

- 玩家墙体与敌人攻击关系。
- 技能费用来源与即时释放规则。
- 敌人击杀、能量拾取和升级循环。
- 波次与 Boss 的基础推进规则。
- 单局胜利和失败条件。

### 不包含

- 具体技能、Perk、怪物和 Boss 内容。
- 费用恢复速度、掉落数量、伤害和生命值等数值。
- Build 组合、进化和突破的详细规则。
- 局外天赋的具体内容。

## 核心概念与状态归属

| 概念/状态 | 归属 | 已确认语义 |
|---|---|---|
| 玩家表现 | 战斗表现 | 玩家角色站在屏幕下方，用于表现防守方；实际受击对象是墙体。 |
| 玩家墙体 | 玩家战斗状态 | 墙体代表玩家生存状态；敌人攻击墙体并造成伤害。 |
| 敌人 | 波次/怪物规则 | 敌人从屏幕上方向下方移动，并根据攻击距离对墙体发动攻击。 |
| 技能费用 | 玩家战斗资源 | 用于主动释放技能，可由时间、技能能量点拾取和技能/被动效果恢复。 |
| 技能费用遮罩 | 战斗 UI 反馈 | 技能费用不足时覆盖技能按钮，并按当前费用距离技能消耗要求的进度旋转；遮罩完全消失时费用正好达到可释放条件。 |
| 技能 | 玩家局内 Build | 玩家触发后立即释放，不选择目标或位置；每个技能使用自己的锁敌逻辑。 |
| 升级经验 | 玩家局内成长 | 敌人死亡后掉落的独立资源；使用与技能能量点相同的拾取逻辑，拾取后推动升级并触发 Perk 三选一。 |
| 技能能量点 | 玩家战斗资源 | 敌人死亡后掉落，用于恢复技能费用；与升级经验使用相同拾取逻辑。 |
| 波次 | 关卡进度 | 怪物刷新由波次组成；每个波次配置一种结束模式，决定是否允许残余敌人与下一波重叠。 |
| Boss 波次 | 关卡进度 | 本质上是普通波次，但在刷新到第 10 个怪物时额外刷新一个 Boss。 |

## 已确认规则

### 防守目标与敌人攻击

1. 玩家表现位于屏幕下方，实际规则对象是一道墙体。
2. 敌人从屏幕上方向下方移动，目标是攻击玩家墙体。
3. 近战攻击与远程攻击使用同一攻击规则，核心差异是攻击距离。
4. 近战敌人到达墙体后，以撞击墙体的方式造成伤害；撞墙后继续存活并持续攻击墙体，直到被玩家消灭。
5. 远程敌人进入自身攻击距离后停止移动，并持续发射弹道攻击墙体，直到被玩家消灭。
6. 墙体被敌人攻击至死亡时，本局失败。

### 技能费用与释放

1. 技能费用随时间自然恢复。
2. 敌人被击败后可以掉落技能能量点；玩家拾取后恢复技能费用。
3. 技能能量点掉落 3 秒后自动拾取；在自动拾取前，玩家可以主动提前拾取。
4. 技能效果和局内升级获得的被动效果可以提供额外费用恢复。
5. 玩家触发技能后立即释放，不需要选择目标或释放位置。
6. 每个技能负责自己的锁敌逻辑。
7. 主动技能没有冷却时间；只要技能费用满足消耗条件，就可以再次释放。
8. 技能费用不足时禁止触发，不提供额外的失败反馈；费用遮罩是唯一的不可释放状态反馈。
9. 费用遮罩根据当前费用距离该技能消耗要求的进度旋转。不同技能费用不同，因此遮罩完成一次费用准备所需的旋转速度不同。
10. 费用遮罩完全消失时，当前费用正好满足技能消耗要求，技能进入可触发状态。
11. 自然恢复、拾取技能能量点、技能效果或被动效果改变当前费用时，费用遮罩同步反映新的准备进度。

### 升级经验与技能费用分离

1. 升级经验和技能费用是两种独立资源。
2. 敌人死亡后分别掉落升级经验和技能能量点。
3. 升级经验与技能能量点使用相同的拾取逻辑：掉落 3 秒后自动拾取，并允许玩家在自动拾取前手动提前拾取。
4. 升级经验用于触发局内升级和 Perk 三选一。
5. 技能费用用于主动释放技能。
6. 技能能量点只恢复技能费用，不直接提供升级经验。

### Perk 三选一暂停

1. 玩家进入 Perk 三选一时，战斗完全暂停。
2. 完全暂停期间，敌人移动与攻击、波次刷新与计时、技能费用自然恢复、掉落物自动拾取计时、已释放技能及其弹道和效果全部停止推进。
3. Perk 三选一界面保持可操作。
4. 玩家完成选择并退出三选一界面后，所有暂停的战斗状态从原进度继续。

### 波次、Boss 与胜利

1. 关卡怪物刷新由多个波次组成。
2. 玩家需要完成关卡内全部波次。
3. 波次支持两种结束模式：
   - **刷新完成型**：本波刷新序列结束后立即允许下一波开始；上一波残余敌人可以与下一波敌人重叠。
   - **清场完成型**：本波刷新序列结束后继续等待，只有场上所有敌人全部死亡时才允许下一波开始。
4. 每个波次必须明确配置一种结束模式。
5. Boss 波次沿用普通波次结构，并在刷新到第 10 个怪物时额外刷新一个 Boss。
6. 首个关卡目标时长为 8 分钟。
7. 首个关卡包含三个 Boss，分别在第 3、6、8 分钟对应的 Boss 波次出现。
8. 第 8 分钟出现最终 Boss。最终波次刷新完成、最终 Boss 死亡且场上所有敌人全部死亡时，本关胜利。

## 单局主循环

1. 战斗开始后，关卡按波次刷新敌人，技能费用随时间恢复。
2. 敌人从上方向玩家墙体推进。
3. 玩家观察敌群和技能费用，选择技能释放时机。
4. 技能立即释放并根据自身规则自动锁定目标。
5. 敌人被击败后分别掉落升级经验和技能能量点。
6. 两种掉落物均在 3 秒后自动拾取，也允许玩家提前手动拾取。
7. 升级经验满足升级条件时进入 Perk 三选一；战斗完全暂停，直到玩家完成选择。
8. 未被击败的敌人继续推进；进入攻击距离后停止推进并持续攻击玩家墙体。
9. 波次根据自身结束模式决定立即进入下一波，或等待清场后进入下一波。
10. 最终波次刷新完成、最终 Boss 和场上其他敌人全部死亡后，本局胜利。
11. 任意时刻墙体死亡，本局失败。

## 玩家反馈与信息需求

- 墙体当前生存状态必须持续可见。
- 技能当前费用、是否可释放和费用变化必须可识别。
- 技能费用不足时显示旋转费用遮罩；遮罩完全消失时表示费用正好达到可释放条件。
- 费用不足时不额外显示触发失败反馈。
- 技能释放后必须从敌人死亡、受击或战场变化中得到明确反馈。
- 当前波次、Boss 即将出现和最终 Boss 阶段必须可识别。
- 获得升级时必须明确进入 Perk 三选一流程。
- Perk 三选一期间必须明确表现战斗处于完全暂停状态。

## 内容与配置语义

- 关卡需要定义目标时长、波次顺序和 Boss 出现时间。
- 波次需要定义怪物刷新序列、结束模式，以及 Boss 是否在第 10 个怪物时追加。
- 怪物需要定义攻击距离、攻击表现和对墙体的伤害规则。
- 技能需要定义费用、锁敌规则和即时释放后的效果；主动技能不配置冷却时间。
- 升级经验和技能能量点需要分别定义掉落条件和资源效果，并共享同一套手动拾取与 3 秒自动拾取规则。
- 战斗 UI 需要根据每个技能的费用要求和当前费用显示旋转费用遮罩；遮罩完全消失表示费用满足。
- 手动拾取的具体手势和视觉表现由后续 UI/UX 规格定义，但升级经验与技能能量点必须使用相同交互。

具体字段名和程序数据结构由下游派生，不属于本文档权威范围。

## 验收标准

- [ ] 玩家墙体死亡时稳定进入失败结算。
- [ ] 近战和远程敌人均能依据攻击距离攻击同一个墙体目标。
- [ ] 远程敌人进入射程、近战敌人撞墙后停止移动并持续攻击，直到被消灭。
- [ ] 技能费用可以通过时间、能量点以及技能/被动效果恢复。
- [ ] 技能能量点在掉落 3 秒后自动拾取，并允许玩家提前手动拾取。
- [ ] 升级经验和技能费用保持独立；两者的掉落物使用相同拾取逻辑。
- [ ] 玩家无需选目标或位置即可立即释放技能，技能自行锁敌。
- [ ] 主动技能没有冷却；费用满足时可以释放，费用不足时禁止触发且不显示额外失败反馈。
- [ ] 不同技能根据自身费用要求显示费用遮罩，遮罩完全消失时费用正好满足。
- [ ] Perk 三选一期间战斗完全暂停，完成选择后从原状态继续。
- [ ] 刷新完成型波次允许残余敌人与下一波重叠，清场完成型波次等待场上敌人全部死亡。
- [ ] 首个关卡在第 3、6、8 分钟触发三个 Boss 遭遇。
- [ ] 最终波次刷新完成、最终 Boss 和场上其他敌人全部死亡后进入胜利结算。
- [ ] 击杀、能量恢复、技能释放、升级三选一和 Build 强化形成可重复循环。
