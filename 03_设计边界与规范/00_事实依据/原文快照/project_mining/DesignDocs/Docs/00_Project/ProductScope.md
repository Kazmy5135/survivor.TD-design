# 产品范围

## 产品定位

- 一款运行在手机端的商业化塔防 Roguelike 手游。
- 本产品是《弹壳特工队》正式授权的衍生产品；原产品由当前设计负责人参与制作。
- 核心战斗采用敌人从屏幕上方向下方推进的塔防结构。
- 玩家需要在被敌人击败前消耗费用激活技能，使用技能消灭敌人。
- 玩家通过击败敌人获得升级机会，并通过 Perk 三选一逐步强化局内 Build。

## 目标玩家与场景

- 运行平台：手机端。
- 核心用户：塔防 Roguelike 用户。
- 扩展用户：喜欢割草、塔防或 Roguelike 玩法的用户，以及《弹壳特工队》的现有用户和流失用户。
- 目标单局时长：8～12 分钟。
- 主要游玩场景：碎片时间游玩。

## 核心体验支柱

1. **割草与技能反馈**：使用技能大量消灭持续推进的敌人；技能使用后的主要反馈来自割草结果，并让玩家清楚感知技能已经产生作用。
2. **目标驱动的局内 Build**：Perk 三选一不只是随机变强，玩家应能识别可追求的 Build 目标；达成目标后获得明确的强度变化和爽感反馈。
3. **Build 多样性**：首版本必须支持多条体验和效果有明显差异的 Build 路线，不能只形成数值大小不同的同质方案。
4. **跨局成长**：通过局外天赋养成提高局内 Build 上限，获得持续成长感。

## 核心重复行为

- 在升级时进行 Perk 三选一。
- 消耗费用使用技能消灭敌人。

## 版本目标

- 建立一个可以完整体验“进入关卡 -> 使用技能战斗 -> 击败敌人升级 -> Perk 三选一强化 -> 形成局内 Build -> 完成或失败结算 -> 局外天赋成长”的首个可玩版本。
- 首版本提供三个相互独立的完整关卡体验。

## 包含范围

- 三个独立关卡及其完整局内规则。
- 八种可供玩家初始选择的主动技能。
- 八种被动技能。
- 主动技能与被动技能相关的进化和突破规则。
- 玩家消耗费用激活技能的战斗机制。
- Perk 三选一及其局内 Build 强化流程。
- 一套独立的天赋局外养成系统。
- 局外天赋效果继承到局内战斗的规则。
- 完整的局内战斗 UI。
- 足以支撑三个关卡体验的怪物与 Boss 设计。

## 非目标

- 超过八种主动技能和八种被动技能的扩展设计。
- 天赋系统之外的其他局外养成系统。
- 商业化功能模块。

## 成功标准

- 三个独立关卡均能形成完整、可结束的战斗体验。
- 玩家可以完成费用消耗、技能使用、击杀升级、Perk 三选一和局内 Build 成长的完整循环。
- 天赋局外成长能够影响后续局内战斗，并体现 Build 上限提升。
- 四项核心体验支柱——割草与技能反馈、目标驱动的局内 Build、Build 多样性和跨局成长——均能在首版本中被明确体验和验证。
- 使用技能后有明确反馈，且主要通过消灭敌人和战场变化体现。
- Perk 选择过程中存在玩家能够识别和追求的 Build 目标；达成目标时产生明确的强度跃迁和爽感反馈。
- 八种主动技能与八种被动技能能够形成多条机制或战斗体验明显不同的 Build 路线。
- Build 多样性是首个 MVP 成功的核心标准之一，不得推迟到 MVP 之后验证。
- 当前 MVP 采用定性成功标准，不以关卡通过率作为产品范围验收指标。
